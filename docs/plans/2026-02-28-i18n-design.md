# i18n: EN/PL language support

## Approach
Two static pages via Astro routing: `/` (EN), `/pl/` (PL). No JS needed for switching.

## Structure

```
src/
  i18n/
    en.ts        - EN UI strings + data (projects, experience, techStack)
    pl.ts        - PL UI strings + data
    types.ts     - shared type definitions
  components/
    Portfolio.astro - shared page template (receives lang data as props)
  pages/
    index.astro  - imports EN, renders Portfolio
    pl/
      index.astro - imports PL, renders Portfolio
  layouts/
    Layout.astro - add `lang` prop to <html>, add hreflang tags
```

## Language switcher
`EN | PL` text toggle in navbar, right side. Active lang highlighted with accent color. Links to `/` or `/pl/`.

## What gets translated
- All UI labels (section titles, nav items, button text, subsection labels)
- Hero bio text
- Project descriptions, highlights, criteria labels
- Experience items
- Education text, interests
- Footer text

## What stays the same
- Tech stack names (Flutter, C#, etc.)
- Company names
- Project titles
- Dates/periods
- Links
